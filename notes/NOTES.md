# How to perform research

Source: https://www.youtube.com/watch?v=sigJwoeU6OI

1. Wikipedia reference section is a good starting point to find sources.
2. Always read and prioritize the primary sources and then go to the secondary sources.
3. Take lots of notes, so as to not have to return to the sources.

Source: https://www.youtube.com/watch?v=4fG5FS3t3ls

1. CVE stands for common vulnerabilities and exposures. (cve.mitre.org)

2. In order to claim finding a security vulnerability, proof is required through a PoC (Proof of Concept).

3. Tools:
    - Static analysis
    - Manual audit
    - Fuzzing

4. Reporting a bug:
    - Report it through private email.
    - Clear reproduction steps (PoC).
    - Explain the bug and why it is a security issue.
    - Disclosure policy. (Example: https://securitylab.github.com/advisories#policy)
